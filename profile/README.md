<aside>

![YogurT-header.png](https://github.com/user-attachments/assets/42dc90b3-adaa-4ae2-95ee-a724d89ea7b8)

**요거티(YogurT)**는 천재교육의 서비스인 ‘AIDT’와 ‘밀크T’에서 영감을 받아 만들어졌으며, 여기서 **T**는 **Textbook(교과서)**와 **Teacher(교사)**를 의미합니다.

‘요거트’처럼 부드럽고 친숙한 학습 도구가 되기를 바라는 마음을 담았습니다.

</aside>

- **YogurT** 는 천재교육의 AIDT(AI 디지털 교과서)를 기반으로 한 차세대 학습 지원 플랫폼입니다.
- 학습 콘텐츠 제공, 학습 진도 추적, 차시별 평가, 그리고 교사와 학생 간의 상호작용을 통합적으로 지원합니다.
- 교사는 자동 수집된 학습 통계를 통해 학급 전체의 학습 현황을 손쉽게 파악하고 효과적으로 지도할 수 있습니다. 또한, 수업 중 학생의 화면을 교사의 화면으로 전환시킬 수 있는 ‘**집중 학습 모드**’ 기능도 제공됩니다.
- 학생은 개별 학습과 평가를 진행하며, 자신의 학습 통계를 실시간으로 확인하고 복습할 수 있습니다.

# 🚦핵심 기능

---

## 수업

- 교과서 단원에 맞는 수업 컨텐츠를 조회 및 재생할 수 있다.
- 학습의 진도율을 저장하여 이어하기 및 빠른 접근이 가능하다.
- 웹소켓을 이용해 현재 접속 인원과 학생의 위치를 모니터링할 수 있다.
- 학생이 보는 화면을 교사의 화면으로 강제이동시키는 집중학습모드를 제공한다.
- 학습 시간을 계산하여 학습 패턴 및 통계를 관리할 수 있다.
- 디지털 교과서와 연계된 실물 교과서의 내용을 PDF로 확인 가능하다.
- 수업, 평가, 교과서 창에 판서가 가능하다.

## 평가

- 교과서 단원별 및 평가유형별 평가를 응시하고 결과를 확인할 수 있다.
- 문항별 응시 시간 및 정오답 여부, 학급의 평균 시간 및 정답률 등의 통계를 확인할 수 있다.
- 교사의 경우 평가창에서 바로 문항의 정답 및 해설을 볼 수 있는 기능을 제공한다.

# 🌿**브랜치 전략 및 깃 컨벤션**

---

## 브랜치 전략

- Git-flow 전략을 기반으로 dep, dev 브랜치와 보조 브랜치 Feat를 운용했습니다.
- dep, deploy, dev, feat 브랜치로 나누어 개발을 하였습니다.
    - (Front 한정) **dep** 브랜치는 배포 단계에서 사용하는 master 브랜치입니다.
    - **deploy** 브랜치는 배포 직전 단계에서 사용하는 브랜치입니다.
    - **dev** 브랜치는 개발 단계에서 git-flow의 master 역할을 하는 브랜치입니다. 
    (Backend의 경우 이 브랜치를 배포단계까지 사용)
    - **Feat** 브랜치는 기능 단위로 독립적인 개발 환경을 위하여 사용하고 merge 후 각 브랜치를 삭제해주었습니다.

## 깃 커밋 컨벤션

| **태그 이름** | **설명** |
| --- | --- |
| feat | 새로운 기능을 추가 |
| fix | 버그 수정 |
| style | CSS 수정 |
| refactor | 코드 리팩토링 |
| conflict | 충돌 해결에만 집중한 경우 |
| chore | 주석이나 사소한 수정 |
| test | 테스트용 코드 |
| deploy | 배포 관련 설정 |
| rename | 파일 혹은 폴더명을 수정하거나 옮기는 작업만인 경우 |
| remove | 코드 및 파일 제거 |

# 🏁 프로젝트 후기

---

## 김노은

이번 프로젝트는 협업이 핵심이었습니다. 역할을 구분하지 않고 함께 기능과 DB를 분석하고 소통하며 개발을 진행했습니다. 

이 과정에서 git 활용 능력이 향상되었고, 여러 기능을 개발하면서 다른 팀원의 코드를 이해하고 프로젝트 전반에 대한 시야를 넓힐 수 있었습니다.

## 김다예

실무 데이터를 이용한 프로젝트를 진행하면서 DB를 분석, 파악하는 능력을 기를 수 있었습니다.

리액트를 이용한 프로젝트를 하면서 상태관리와 컴포넌트 분리의 중요성을 느낄 수 있었습니다.

## 김성우

이번 프로젝트는 실제 서비스를 클론코딩했던 만큼 실제 프로젝트의 완성도나 규모를 느낄 수 있었고 주니어 개발자로서 시야를 넓힐 수 있는 기회였습니다.

또한 학습창은 여러가지 상태와 잦은 비동기 데이터를 다루기 때문에 상태관리와 캐싱으로 성능을 개선을 해본 좋은 경험이 되었습니다.

## 김희진

이번 프로젝트는 기존의 DB 데이터를 분석, 맞춰가는 과정이 어려웠지만, 이를 통해 실무와 유사한 경험을 할 수 있었습니다. 리액트를 배우면서 구현했는데 시간상 개선하거나 넣지 못한 부분이 있어 아쉬움이 남습니다. API를 활용해 백엔드와 프론트엔드 간 데이터 흐름을 구현하면서 서비스 흐름에 대한 이해도를 높일 수 있었습니다. 

프로젝트에서 팀장 역할이었는데 제 기능 구현에 집중하느라, 팀원들을 충분히 챙기지 못한 점이 가장 아쉬웠고, 팀워크와 리더의 책임감을 다시금 느꼈습니다.

## 이영주

이번 프로젝트를 통해 WebSocket을 이용한 실시간 데이터 통신을 다루며 Socket에 대한 이해를 높일 수 있었고,
규모 있는 데이터를 DB에서부터 클라이언트까지 정제하여 렌더링 하면서 실무에 대한 경험을 쌓을 수 있는 기회가 되었습니다.
